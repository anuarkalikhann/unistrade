# Как развернуть проект на Vercel

Самый простой и надежный способ сделать сайт доступным для всех — использовать **Vercel**. Это платформа от создателей Next.js.

## Способ 1: Через GitHub (Рекомендуемый)
Этот способ лучше всего, так как сайт будет автоматически обновляться каждый раз, когда вы сохраняете изменения в GitHub.

### 1. Сохраните изменения в Git
Откройте терминал в папке проекта и выполните команды:
```bash
git add .
git commit -m "Готов к деплою"
```

### 2. Создайте репозиторий на GitHub
1. Зайдите на [github.com](https://github.com) и создайте новый репозиторий (New Repository).
2. Следуйте инструкциям GitHub, чтобы привязать вашу папку к репозиторию. Обычно это команды:
```bash
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/НАЗВАНИЕ_РЕПОЗИТОРИЯ.git
git push -u origin main
```

### 3. Подключите Vercel
1. Зайдите на [vercel.com](https://vercel.com) и войдите (можно через GitHub).
2. Нажмите **"Add New..."** -> **"Project"**.
3. Выберите только что созданный репозиторий из списка (Import).
4. Нажмите **"Deploy"**.
5. Готово! Vercel выдаст вам ссылку вида `your-project.vercel.app`.

---

## Способ 2: Через командную строку (Быстрый)
Если вы не хотите возиться с GitHub, можно "закинуть" сайт прямо с компьютера.

1. В терминале в папке проекта напишите:
```bash
npx vercel
```
2. Вас попросят войти (нажмите Enter, откроется браузер).
3. Далее отвечайте на вопросы в терминале (обычно на всё можно нажимать `Enter`):
   - Set up and deploy? **Yes**
   - Which scope? **(Ваш аккаунт)**
   - Link to existing project? **No**
   - Project name? **(Нажмите Enter)**
   - Directory? **(Нажмите Enter)**
   - Want to modify settings? **No**

4. После завершения он выдаст ссылку `Production: https://...` — это и есть ваш сайт.

## Важно
Перед деплоем мы уже проверили, что проект успешно собирается (`npm run build`). Если на Vercel возникнут ошибки, они обычно связаны с переменными окружения или настройками TypeScript, но сейчас проект чист и должен загрузиться без проблем.
